# Integration Touchpoints — Handoff to IT Integration Workstream

Points where workflows depend on integrations between Optro and other systems (HR systems, GRC tools, audit committee systems, ticketing systems, etc.).

**Format:** Newest at top. Describe the workflow need; the IT team designs the integration.

**Ownership:** IT integration workstream owns integration design. Our role ends at surfacing the touchpoint.

## Criticality key

- **Blocking** — workflow cannot function without this integration at go-live
- **Important** — workflow can function manually but at significant cost
- **Nice-to-have** — automates something currently manual, but no regression if deferred

## Status key

- 🟡 Flagged — surfaced, awaiting IT response
- 🟢 Acknowledged — IT has confirmed receipt
- ✅ Planned / scoped — IT has committed to build
- ❌ Out of scope / deferred — surface as a known limitation
